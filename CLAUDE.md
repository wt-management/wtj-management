# wtj-management — 원텍 일본법인 경영관리

이 저장소는 **`index.html` 파일 하나**가 사이트 전체다(빌드 없음, HTML 하나에 CSS·JS 전부 내장). 일본법인의 매출·수금·미수·자금일보·거래처·소모품·대시보드를 한 곳에서 관리한다.

- 라이브 주소: https://wt-management.github.io/wtj-management/
- 데이터는 코드가 아니라 Supabase에 있음(아래). 이 저장소를 고쳐도 데이터는 안 지워진다.

---

## ⚠️ 반드시 지킬 규칙

1. **수정 → main에 바로 반영(직접 push).** 별도 PR 없이 main에 커밋·푸시하면 몇 분 뒤 라이브에 자동 반영된다.
   ```
   git add -A
   git commit -m "무엇을 왜 고쳤는지"
   git push
   ```
   - ⚠️ **push 전에 반드시 확인**: 브라우저에서 열어 동작 OK, 콘솔 에러 0. 바로 라이브로 나가므로 깨진 채 올리면 사이트가 깨진 채 보인다.
   - 잘못 올렸으면 즉시 되돌리기: `git revert HEAD && git push`.

2. **`index.html`만 수정한다.** `.nojekyll`, `README.md`, `.github/` 등은 건드리지 말 것. `.nojekyll`은 특히 삭제 금지(지우면 사이트 빌드가 깨진다).

3. **비밀키(관리자 키) 커밋 금지.** `sb_secret_...` 또는 service_role 키는 절대 넣지 말 것. 사이트에 이미 들어있는 `sb_publishable_...` 키는 공개용이라 정상.

4. **공용 셸은 건드리지 말 것.** 로그인 · 권한 · 부서별 메뉴(MENU_ROLES) · 자동 로그아웃 · 사이트 이동 · 비밀번호 변경 관련 코드는 전 사이트 공통이라 오너 영역이다.

## 데이터 (Supabase)

- 테이블: **`wtj_data`** (key / data jsonb)
  - **key = `finance`** — 현재 사이트의 **단일 진실원(single source of truth)**. 매출일보·자금일보 기반의 매출·수금·미수·소모품·거래처가 전부 여기서 계산됨.
  - key = `main` — 구(舊) 매출엑셀 파이프라인 잔재. 일부 화면의 폴백으로만 남아 있음. 새 기능은 항상 `finance`를 쓴다.
- **주의 — 마지막 저장이 이김:** 저장 시 블롭 전체를 덮어쓴다. 동시 편집 시 유실 가능.
- **로컬 테스트 안전:** localhost로 열면 저장이 안 되고 샘플/읽기 전용으로 동작한다.

### 매출 집계 기준 (절대 함부로 바꾸지 말 것 — 회계와 맞춰둔 규칙)
- **매출인식 = 매출기준일(recog)**. 미승인기기(본사송금〇)는 **선적일이 입력돼야만** 인식(미선적은 제외).
- **금액 = 세전(net, 소비세 제외)**, **수수료(Commission) 행은 매출에서 제외**.
- 대시보드 "제품/소모품 **수주** 상세"는 예외적으로 **수주일(발주일)** 기준(매출인식과 의도적으로 다름).
- 이 로직을 바꾸면 임원 보고 숫자가 틀어진다. 관련 요청이 오면 오너에게 먼저 확인.

## 수정한 뒤 반드시 확인
- 브라우저에서 실제로 열어 동작 확인, **콘솔 에러 0개**
- 한국어 / 日本語 토글 둘 다 안 깨지는지(자동번역 내장)
- JS 문법: 큰 수정 후 `node --check`로 추출 스크립트 검사 권장
- 그 다음 커밋 → push

## 자주 하는 작업 위치 (index.html 안에서 검색)
- 제목·라벨·색상 → 해당 텍스트나 CSS 변수(`--...`) 검색
- 새 메뉴(페이지) 추가 → nav 항목 + 페이지 div + pageTitles + showPage 분기
- 번역 → `T`(한/일 라벨 딕셔너리)

막히면 오너에게 물어보고, 큰 구조 변경·집계기준 변경은 먼저 오너 확인을 받는다.
